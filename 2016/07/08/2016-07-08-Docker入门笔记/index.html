Docker 入门笔记：从零到部署

2016-07-08 技术分享 #Docker - 容器化 - DevOps

什么是 Docker

Docker 是一个容器化平台，可以将应用及其依赖打包在一起。

容器 vs 虚拟机

容器共享宿主机内核，比虚拟机更轻量、启动更快。

`ash

启动一个 nginx 容器

docker run -d -p 8080:80 nginx
`

核心概念

理解镜像、容器和仓库三者的关系是使用 Docker 的关键。

镜像 Image

镜像是容器的模板，定义了运行环境和应用代码。

容器 Container

容器是镜像的运行实例，可以启动、停止、删除。

仓库 Registry

Docker Hub 是最大的公共镜像仓库，也可以搭建私有仓库。

实战：部署一个 Node.js 应用

用一个完整的例子串联所有概念。

编写 Dockerfile

dockerfile FROM node:20-alpine WORKDIR /app COPY . . RUN npm ci EXPOSE 3000 CMD ["node", "index.js"]

构建和运行

ash docker build -t my-app . docker run -d -p 3000:3000 my-app

为什么需要工作流

多人协作时，统一的 Git 工作流可以减少冲突，提高效率。

常见的三种工作流

  1. Git Flow — 适合有固定发布周期的项目
  2. GitHub Flow — 适合持续部署的项目
  3. GitLab Flow — GitHub Flow 的增强版

Git Flow 详解

最经典的分支模型，定义了严格的分支角色。

主要分支

  • main — 生产就绪代码
  • develop — 开发主线
  • eature/* — 功能开发

elease/* — 发布准备

  • hotfix/* — 紧急修复

功能开发流程

从 develop 创建 feature 分支，完成后合并回 develop。

`ash
git checkout develop
git checkout -b feature/user-login

… 开发 …

git checkout develop
git merge feature/user-login
`

Commit 规范

好的 commit message 是可维护代码库的基础。

Conventional Commits

格式：():

常用 type：feat、fix、docs、refactor、test。

规范化的 commit 可以自动生成 changelog 和版本号。
