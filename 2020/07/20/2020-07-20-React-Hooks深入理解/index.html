React Hooks 深入理解：从入门到自定义

2020-07-20 技术分享 #React - Hooks - 前端

为什么需要 Hooks

Hooks 让函数组件也能拥有状态和生命周期能力。

Class 组件的问题

  1. 复杂组件难以拆分
  2. this 绑定困扰
  3. 逻辑复用困难

Hooks 的优势

逻辑可以按关注点分离，而不是按生命周期方法分离。

核心 Hooks 详解

掌握这几个 Hook，就能应对大部分场景。

useState

jsx const [count, setCount] = useState(0);

更新函数可以接收前一个值：setCount(prev => prev + 1)

useEffect

处理副作用：数据请求、订阅、DOM 操作。

jsx useEffect(() => { document.title = Count: ; return () => { /* 清理逻辑 */ }; }, [count]);

useRef

在多次渲染之间保持引用不变。

自定义 Hook

把可复用的逻辑抽成自定义 Hook，是 Hooks 的精髓。

useWindowSize 示例

jsx function useWindowSize() { const [size, setSize] = useState({w: 0, h: 0}); useEffect(() => { const fn = () => setSize({w: window.innerWidth, h: window.innerHeight}); window.addEventListener('resize', fn); return () => window.removeEventListener('resize', fn); }, []); return size; }

自定义 Hook 的命名必须以 use 开头。
