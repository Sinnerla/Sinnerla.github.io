Docker 容器化部署 Hexo 博客

2026-06-01 技术分享 #Docker #Hexo #部署 #Nginx

为什么用 Docker 部署博客

使用 Docker 部署 Hexo 博客可以实现环境一致性，避免「在我机器上能跑」的问题。

Docker 部署的优势

  • 环境隔离，不依赖宿主机 Node.js 版本
  • 一键部署，配置即代码
  • 方便迁移和扩展

整体架构

整个部署流程分为三个阶段：构建、生成和运行。

编写 Dockerfile

Dockerfile 是多阶段构建的核心，以下是完整配置。

第一阶段：构建阶段

使用 Node.js 官方镜像安装依赖并生成静态文件。

第二阶段：运行阶段

使用 Nginx 提供静态文件服务。

Nginx 配置优化

合理的 Nginx 配置可以提升博客的访问性能。

启用 Gzip 压缩

设置缓存策略

静态资源设置长期缓存，利用版本哈希自动刷新。

使用 Docker Compose 编排

Docker Compose 可以简化多容器部署的管理。

生产环境建议配合 Nginx Proxy Manager 或 Traefik 实现 HTTPS 自动续签。
